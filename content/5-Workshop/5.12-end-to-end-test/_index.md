Now that the entire backend and frontend infrastructure is deployed, we will run an end-to-end test of the entire system flow via the WebUI.

#### 1. Access the WebUI Dashboard

Open your browser and navigate to your WebUI URL (from CloudFront or S3 static website). You will see the Admin Dashboard with an overview of total test runs, Pass/Fail rates, and weekly trends.

![Dashboard](/images/5-Workshop/5.12-end-to-end-test/1-dashboard.jpg)

#### 2. Manage Test Suites

The next step is to upload your Playwright test script.

- In the left menu, select **Test Suites**.
- Click the **Add Test Suite** button in the top right corner.

![Test Suites Management](/images/5-Workshop/5.12-end-to-end-test/2-test-suites.jpg)

- In the modal, fill in the following information:
  - **File name:** The display name of the file (e.g., `login.spec.js`)
  - **Website:** The target website
  - **Description:** A short description of the test script
- Click **Choose File** to select the `.spec.js` file from your local machine and click **Upload**. This file will be uploaded to Amazon S3.

![Add Test Suite](/images/5-Workshop/5.12-end-to-end-test/3-add-test-suite.jpg)

#### 3. Manual Test Trigger

Now we will manually trigger a test execution.

- In the left menu, select **Manual Trigger**.

![Manual Trigger](/images/5-Workshop/5.12-end-to-end-test/4-manual-trigger.jpg)

- Fill in the parameters to initialize the test session:
  - **Target URL:** Enter the website URL (e.g., `https://the-internet.herokuapp.com/`)
  - **Environment:** Select the environment (e.g., `Staging`)
  - **Test Suite:** Select the test file you uploaded in Step 2.
- Click the **Trigger Test Now** button.

![Manual Trigger Information](/images/5-Workshop/5.12-end-to-end-test/5-manual-trigger-filled.jpg)

The system will instantly send a message to the SQS queue, and ECS Fargate will pull the Docker Image to execute the script.

![System Initialization](/images/5-Workshop/5.12-end-to-end-test/6-initializing.jpg)

Once the message is successfully sent, the screen will display **"Request queued successfully!"**. The test session is now running in the background on ECS.

![Request Queued](/images/5-Workshop/5.12-end-to-end-test/7-queued.jpg)

#### 4. View Test History and Details

To monitor the results of the executions:

- Go to the **History** menu. Here you will see a list of test runs along with their status (`Pass` or `Fail`), duration, and trigger source. All this data is persistently stored in DynamoDB.

![Test History](/images/5-Workshop/5.12-end-to-end-test/8-history.jpg)

- Click the **Detail** button on a row to view the execution logs. This interface pulls logs directly from **CloudWatch Logs**, allowing you to monitor the Playwright test progress.

![Test Detail](/images/5-Workshop/5.12-end-to-end-test/9-history-detail.jpg)

#### 5. Receive Results and Email Reports

After the test finishes, the Post-processing Lambda function will automatically fetch the results, generate an AI summary, and send an email via Amazon SES.

- Check your inbox. You should receive an email containing a summary of the result (Pass/Fail) and a Generative AI evaluation.

![Email Report](/images/5-Workshop/5.12-end-to-end-test/10-email-report.jpg)

- In the email, click the **"View detailed report"** link. This will open the comprehensive HTML report generated by Playwright, hosted securely on the S3 report bucket.

![HTML Report](/images/5-Workshop/5.12-end-to-end-test/11-html-report.jpg)

#### 6. Additional Configurations

- **Test Schedule:** You can also set up automated cron schedules for periodic test scripts using Amazon EventBridge in the **Test Schedule** menu.
  ![Test Schedule](/images/5-Workshop/5.12-end-to-end-test/12-schedule.jpg)

- **Email Configuration:** Manage the list of recipients who will receive test result notifications in the **Email Configuration** menu.
  ![Email Configuration](/images/5-Workshop/5.12-end-to-end-test/13-email-config.jpg)

Congratulations! You have successfully configured and run the entire Serverless Playwright Automation Framework on AWS.
